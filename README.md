# YanPM 语言驱动项目管理 MVP

这是一个本地可运行的 Web App MVP，用来验证“通过自然语言交流自动进行项目管理”的核心闭环。当前形态是单页 Web 应用 + 可选本地 Node 后端；正式产品可以演进为云端 Web、PWA、企业微信/微信生态入口或小程序。

## 已实现能力

- 项目工作台：健康度、阻塞、临期任务、开放风险和 AI 下一步建议
- 多项目组合：AI 问答建项、切换、复制、归档项目，并查看跨项目总览
- 悬浮全局语言入口：固定在页面底部，任何页面都可无感输入、语音、上传附件或发送，默认归入当前项目，也可识别文本中的项目名自动归属
- AI 建项向导：支持项目模板，收集目标、时间、团队、范围、风险和成功标准，自动生成项目管理框架
- AI 运行设置：默认本地模拟，也可配置 OpenAI、DeepSeek、通义千问、Kimi、智谱、豆包、千帆、混元、MiniMax、硅基流动等接口
- 语言输入：通过底部悬浮入口粘贴会议纪要、聊天记录、日报，自动提取候选变更
- AI 问答：询问项目状态、风险、待办、负责人和简报
- 待确认队列：任务、风险、决策、状态更新先确认再写入项目
- 任务管理：支持看板和甘特图两种模式，支持父子任务、依赖、验收标准、置信度和 AI 推断标记
- 依赖任务选择：编辑任务和候选任务时可从已有任务中多选依赖项
- 微信/企业微信身份认证：支持微信开放平台、企业微信和本地开发实名模式，用于审计追踪
- 审计追踪：记录登录、任务、风险、候选变更、导入导出、归档恢复等关键操作，并支持导出 CSV
- AI 拆解引擎：把粗工作包拆成可监督的子任务，缺少负责人、时间或验收标准时自动生成追问
- 健康度解释：展示延期、阻塞、高风险、负责人缺失、验收缺失、待确认和久未更新等扣分原因
- 风险管理：创建、编辑、关闭、删除风险
- 团队跟进：按负责人聚合负载、阻塞、延期和跟进消息
- 里程碑追踪：在工作台展示关键节点、负责人、截止时间和自动进度
- 智能汇报：自动生成项目周报、管理层摘要和风险升级摘要
- 项目记忆：记录每次确认后的来源和变更历史
- 数据导入导出：使用 JSON 在本地保存项目数据
- 语音听写与录音入口：支持浏览器听写，录音文件可先作为来源记录，后续接入真实转写服务

## 使用方式

推荐使用本地后端运行：

```bash
npm start
```

然后打开：

```text
http://127.0.0.1:8787
```

这样项目数据会保存到 `data/state.json`，AI 请求也可以通过后端代理转发，避免浏览器 `file://`、CORS 和本地存储限制。

也可以直接在浏览器打开 `index.html` 体验静态版。静态版会优先使用浏览器本地存储；如果当前环境禁止本地存储，会自动退回到当前页面会话内存。

## 部署方式

第一版建议按“Web 应用 + Node 后端”部署，不要只部署静态页面。原因是微信/企业微信登录回调、AI 代理、服务端 API Key、审计日志都需要后端。

免费测试部署可以使用 Netlify 或 Render：

- Netlify：适合已有 Netlify 账号的测试发布，已提供 `netlify.toml`、Netlify Functions 和 Netlify Blobs 状态保存。详细步骤见 `DEPLOY_NETLIFY.md`。
- Render：适合直接运行 Node 后端的测试发布，已提供 `render.yaml`。详细步骤见 `DEPLOY_RENDER.md`。

最小可发布部署：

1. 准备一个 HTTPS 域名，例如 `https://yanpm.example.com`
2. 在服务器设置环境变量，参考 `.env.example`
3. 生产环境设置 `HOST=0.0.0.0`
4. 设置 `YANPM_PUBLIC_URL=https://yanpm.example.com`
5. 设置 `YANPM_ENABLE_CODEX_TEST=0`
6. 配置 AI API Key 到服务端环境变量，不放浏览器里
7. 配置微信开放平台或企业微信回调地址
8. 运行发布检查：

```bash
npm run release:check
```

9. 启动服务：

```bash
node server.mjs
```

微信开放平台回调地址：

```text
https://yanpm.example.com/api/auth/wechat/callback
```

企业微信回调地址：

```text
https://yanpm.example.com/api/auth/wecom/callback
```

容器部署：

```bash
docker build -t yanpm-web .
docker run -p 8787:8787 --env-file .env yanpm-web
```

当前后端仍使用 `data/state.json` 作为本地持久化，适合内测和单机试点。正式团队版应把项目数据、用户、组织、权限和审计日志迁移到数据库，并限制审计导出权限。

推荐体验流程：

1. 点击顶部“新建项目”
2. 按 AI 建项向导回答项目目标、时间、团队、范围、风险和成功标准
3. 生成项目后进入“工作台”查看自动创建的里程碑、任务和风险
4. 进入“项目组合”查看所有并行项目
5. 使用顶部项目切换器切换当前项目
6. 在底部悬浮输入框输入会议纪要、聊天记录或日报
7. 点击发送
8. 进入“待确认”
9. 确认候选变更
10. 进入“审计”查看操作记录，或导出审计 CSV

底部悬浮入口始终固定显示。它默认把输入归入当前项目；如果内容里明确提到另一个项目名称，会自动切换到对应项目再处理。

## v0.2 项目管理闭环

当前版本已经加入更接近项目经理工作方式的闭环：

- 语言输入会优先拆解工作包，而不是只生成一条粗任务
- 任务可记录父任务、依赖任务、开始时间、截止时间和验收标准
- 依赖任务可直接从已有任务列表中多选，依赖未完成时会参与阻塞风险判断
- 待确认变更按任务、追问、风险、里程碑、状态更新和决策分组
- AI 推断出的负责人、时间等字段会显示置信度和推断提示
- 甘特图按层级展示任务，工作包会根据子任务自动汇总进度
- 工作台健康度会解释扣分原因，便于判断下一步该跟进什么

## AI 设置

顶部齿轮按钮可以配置 AI 运行方式：

- Codex 临时测试：开发期通过本地后端调用当前机器的 Codex CLI 登录态，不需要在浏览器填写 API Key；正式发布前应移除此模式
- 本地模拟：不调用外部 API，适合演示流程
- OpenAI Responses API：填写 API Key、Base URL 和模型后调用真实模型
- OpenAI-compatible Chat Completions：适配兼容 OpenAI `/chat/completions` 的服务

“模型供应商”下拉框会自动填入常用供应商的 Base URL、运行模式和推荐模型，包括：

- Codex 临时测试
- OpenAI / ChatGPT
- DeepSeek 深度求索
- 阿里云通义千问 DashScope
- Moonshot / Kimi
- 智谱 GLM
- 火山方舟 / 豆包
- 百度智能云千帆
- 腾讯混元
- MiniMax
- 硅基流动 SiliconFlow
- 自定义 OpenAI-compatible 服务

国内模型平台的可用模型名、Endpoint ID 和区域会随账号开通情况变化。设置页里的 Base URL 和模型都可以继续手动修改，建议以对应平台控制台显示为准。

Moonshot/Kimi 的 `kimi-k2.6` 对 temperature 参数有限制，应用会自动使用兼容值，避免出现 `invalid temperature: only 1 is allowed for this model`。

API Key 仅保存在当前浏览器的本地存储中。正式产品应改为后端代理，避免在浏览器中暴露密钥。

本地后端代理支持用环境变量保存密钥：

```bash
AI_API_KEY=你的密钥 npm start
```

使用本地后端时，前端如果没有填写 API Key，会优先使用后端的 `AI_API_KEY`，也兼容旧的 `OPENAI_API_KEY`。

Codex 临时测试模式需要从本地服务地址打开应用：

```bash
npm start
```

然后访问 `http://127.0.0.1:8787`。后端会调用本机 `codex exec`，默认超时 60 秒；可以用 `CODEX_BIN` 指定 Codex 路径，用 `YANPM_ENABLE_CODEX_TEST=0` 关闭该临时通道。

全局语言入口支持键盘输入、文本导入、附件、照片、录音文件和麦克风。麦克风会优先使用浏览器语音听写，识别文本会直接写入输入框；如果浏览器不支持语音转文字，会退回到麦克风录音来源记录。

如果浏览器或 `file://` 环境禁止本地存储，应用会自动退回到当前页面会话内存，功能仍可体验，但刷新后不会保留数据。

## 多端适配

当前版本是响应式 Web App，已适配桌面、平板和手机宽度：桌面为侧边栏布局，窄屏会折叠为横向导航，工作台、团队、汇报、记忆等多列内容会自动变为单列。当前还不是正式 PWA，也没有云端多设备同步；多终端同步需要后续接入后端账号、组织、权限和数据库。

## 登录与审计

当前版本已经预留微信开放平台网站应用登录和企业微信登录。没有配置 AppID/Secret 时，会保留本地开发实名模式，方便继续体验和审计。

推荐正式发布流程：

1. 用户使用微信扫码或企业微信授权登录
2. 后端完成 OAuth 回调校验
3. 应用要求用户补充真实姓名、手机号或企业邮箱、组织/团队
4. 审计日志优先显示真实姓名，同时保留微信/企微身份绑定信息

审计追踪会记录关键操作的时间、操作者、项目、动作和详情，包括任务编辑、风险处理、候选变更确认/忽略/合并、导入导出、项目切换、归档恢复、登录退出等。审计记录当前随项目数据保存在本地；正式产品应写入后端不可篡改日志。

服务端环境变量见 `.env.example`。第一版发布前请按 `RELEASE_CHECKLIST.md` 完成检查，尤其要关闭 `YANPM_ENABLE_CODEX_TEST`，并把 AI API Key 留在后端。
